Repository Info (sampling):

Information on the repositories (repo and git):

The Android project uses 'repo' as a repository tool (repo is built on top of git; see http://source.android.com/source/version-control.html ). If you want to learn about the repository for Android, see http://source.android.com/source/life-of-a-patch.html. Basically, the roles are 'reviewer', 'verifier', and 'approver'. Also, there is an 'owner' of a bug, change, or project.

Patch Approval people and process:

There is a huge community here; it is so big that it has its own pages, starting with http://source.android.com/community/index.html, where you will find the rules of participation, as well as a link to the forum set, discussion lists and IRChat.
